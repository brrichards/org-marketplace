# FluidFramework Conventions

When writing code for FluidFramework:

- Use TypeScript strict mode
- Follow the existing import ordering conventions
- Use ESLint and Prettier configurations from the repo
- Prefer `interface` over `type` for object shapes
- Use `readonly` where possible
- Document public APIs with TSDoc comments
